Q: is there any kind of maturity letter mentioned in the PDFs? If so, show it and also show the source. 

A: 
No exact "maturity ladder" is mentioned in the three PDFs.

The closest match is an "AI MATURITY [AGENT USE]" breakdown in *The State of AI-Native Engineering in 2026* by Augment Code. It lists four maturity/use categories:

- Specific tasks
- Workflows with engineer oversight
- Multiple parallel tasks
- Autonomous end-to-end

Source: `state-of-ai-native-engineering-2026.pdf`, page 4, Survey Details / Who responded.

There is also a staged adoption chart in *The LeadDev Engineering Leadership Report 2026*. It asks: "Which of the following best describes the stage of your engineering organization's adoption of AI-powered coding tools and agents?"

Source: `ENGINEERING_LEADERSHIP_REPORT_2026_FINAL.pdf`, page 14, "AI adoption is widespread but poorly measured".

Q: if there is a maturity letter mentioned or some other thing, show the percentage of companies that they interviewed. Also show the source. 

A: 
For the Augment Code "AI MATURITY [AGENT USE]" breakdown, the report surveyed 219 engineering leaders. The PDF shows counts, not percentages, so the percentages below are calculated from 219 respondents:

- Specific tasks: 67 respondents, about 31%
- Workflows with engineer oversight: 64 respondents, about 29%
- Multiple parallel tasks: 59 respondents, about 27%
- Autonomous end-to-end: 28 respondents, about 13%

Source: `state-of-ai-native-engineering-2026.pdf`, page 4, Survey Details / Who responded.

For the LeadDev adoption-stage chart, the PDF provides percentages directly:

- Adopted by all teams for all use cases: 12%
- Widely adopted by many teams for many use cases: 42%
- Selectively adopted for a few use cases: 22%
- Proof-of-concept, experimenting with tools and pilot projects: 16%
- Exploratory, researching tools and evaluating use cases: 8%
- No plans to use AI tools and models: 1%

Source: `ENGINEERING_LEADERSHIP_REPORT_2026_FINAL.pdf`, page 14, "AI adoption is widespread but poorly measured".
